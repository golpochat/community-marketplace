## Document template

Every feature doc includes:

1. **Functional requirements** — what the system must do
2. **Non-functional requirements** — performance, security, availability
3. **User flows** — Mermaid diagrams
4. **Edge cases** — failure modes and handling
5. **Acceptance criteria** — testable checklist
